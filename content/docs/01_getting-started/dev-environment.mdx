## 1. Code Editor

We recommend **Visual Studio Code** (VS Code). It's free, powerful, and used by most open source projects.

### Install VS Code

<div className="bg-neutral-800/50 border border-neutral-700 rounded-lg p-6 mb-6">

**macOS:**
```bash
brew install --cask visual-studio-code
```

**Linux (Ubuntu/Debian):**
```bash
sudo apt install software-properties-common apt-transport-https wget
wget -q https://packages.microsoft.com/keys/microsoft.asc -O- | sudo apt-key add -
sudo add-apt-repository "deb [arch=amd64] https://packages.microsoft.com/repos/vscode stable main"
sudo apt install code
```

**Windows:**

Download from [code.visualstudio.com](https://code.visualstudio.com)

</div>

| Extension | Purpose |

## 2. Terminal Setup

A good terminal experience makes everything easier.

### macOS

The default Terminal is fine, but consider:

- **iTerm2** — More features, better customization
- **Warp** — Modern, AI-assisted terminal

```bash
# Install iTerm2
brew install --cask iterm2
```

### Linux

Most distributions come with a good terminal. Popular options:
- **GNOME Terminal** (default on Ubuntu)
- **Alacritty** (GPU-accelerated, fast)
- **Kitty** (GPU-accelerated, feature-rich)

### Windows

Use **Windows Terminal** (included in Windows 11) with WSL2:

```powershell
# Install WSL2
wsl --install

# Set Ubuntu as default
wsl --set-default Ubuntu
```

<Warning title="Windows Users">
Most open source projects assume a Unix-like environment. Use WSL2 for the best experience on Windows.
</Warning>

### Shell Customization

Consider upgrading your shell:

```bash
# Install Oh My Zsh (makes zsh more powerful)
sh -c "$(curl -fsSL https://raw.github.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"
```

## 3. Git Configuration

### Install Git

<div className="bg-neutral-800/50 border border-neutral-700 rounded-lg p-6 mb-6">

**macOS:**
```bash
brew install git
```

**Linux:**
```bash
sudo apt install git
```

**Windows (WSL):**
```bash
sudo apt install git
```

</div>

### Configure Git Identity

```bash
# Set your name (use your real name)
git config --global user.name "Your Name"

# Set your email (use your GitHub email)
git config --global user.email "your.email@example.com"

# Set default branch name
git config --global init.defaultBranch main

# Set default editor
git config --global core.editor "code --wait"

# Enable colors
git config --global color.ui auto
```

### Set Up SSH Keys

SSH keys let you authenticate with GitHub without entering your password every time.

<Steps>
<Step number={1} title="Generate SSH Key">
```bash
ssh-keygen -t ed25519 -C "your.email@example.com"
```
Press Enter to accept the default location. Optionally add a passphrase.
</Step>

<Step number={2} title="Start SSH Agent">
```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```
</Step>

<Step number={3} title="Add Key to GitHub">
```bash
# Copy your public key
cat ~/.ssh/id_ed25519.pub
```
Then go to GitHub → Settings → SSH and GPG keys → New SSH key, and paste your key.
</Step>

<Step number={4} title="Test Connection">
```bash
ssh -T git@github.com
```
You should see: "Hi username! You've successfully authenticated..."
</Step>
</Steps>

## 4. Language Runtimes

Install the languages you'll be working with. Here are the most common:

### Node.js

Use **nvm** (Node Version Manager) to manage Node.js versions:

```bash
# Install nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Restart terminal, then install Node
nvm install --lts
nvm use --lts

# Verify
node --version
npm --version
```

### Python

Use **pyenv** to manage Python versions:

```bash
# Install pyenv
curl https://pyenv.run | bash

# Add to your shell config (.zshrc or .bashrc)
export PATH="$HOME/.pyenv/bin:$PATH"
eval "$(pyenv init -)"

# Install Python
pyenv install 3.12
pyenv global 3.12

# Verify
python --version
```

### Rust

```bash
# Install Rust via rustup
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Verify
rustc --version
cargo --version
```

### Go

```bash
# macOS
brew install go

# Linux
wget https://go.dev/dl/go1.22.0.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.22.0.linux-amd64.tar.gz
export PATH=$PATH:/usr/local/go/bin

# Verify
go version
```

## 6. Docker (Optional but Recommended)

Many projects use Docker for consistent development environments.

### Install Docker

**macOS:**
```bash
brew install --cask docker
```

**Linux:**
```bash
# Install Docker Engine
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# Add user to docker group (to run without sudo)
sudo usermod -aG docker $USER
```

### Verify Installation

```bash
docker --version
docker run hello-world
```

## Common Issues

<CardGrid>
<Card title="SSH Key Not Working" icon={<AlertTriangle className="h-5 w-5 text-yellow-400" />}>
- Make sure ssh-agent is running
- Check the key is added: `ssh-add -l`
- Verify key is in GitHub settings
</Card>

<Card title="Permission Denied" icon={<AlertTriangle className="h-5 w-5 text-yellow-400" />}>
- Don't use `sudo` with package managers
- Check file permissions
- Use nvm/pyenv instead of system packages
</Card>

<Card title="Command Not Found" icon={<AlertTriangle className="h-5 w-5 text-yellow-400" />}>
- Restart your terminal
- Check PATH in your shell config
- Verify installation completed
</Card>

<Card title="WSL Issues" icon={<AlertTriangle className="h-5 w-5 text-yellow-400" />}>
- Run updates: `wsl --update`
- Ensure WSL2: `wsl --set-version Ubuntu 2`
- Check Windows features are enabled
</Card>
</CardGrid>
