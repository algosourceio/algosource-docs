Your GitHub profile is your open source resume. Maintainers will check it before reviewing your PRs. Make it count.
</Callout>

## Why Your Profile Matters

When you submit a PR, maintainers often click on your profile to answer:

- Is this person legitimate?
- Have they contributed to other projects?
- Are they likely to follow through?
- Will they be pleasant to work with?

A good profile answers these questions positively.

## Essential Profile Elements

### 1. Profile Picture

<div className="bg-neutral-800/50 border border-neutral-700 rounded-lg p-6 mb-6">

**Do:**
- Use a clear photo of your face OR
- Use a recognizable avatar/logo you use consistently

**Don't:**
- Use the default GitHub avatar
- Use memes or offensive images
- Change it frequently

<Callout type="tip" title="Consistency Matters">
Use the same avatar across GitHub, Twitter, Discord, and your blog. It builds recognition.
</Callout>

</div>

### 2. Display Name

- Use your real name OR
- Use a consistent pseudonym you use everywhere

**Examples:**
- ‚úÖ "Sarah Chen"
- ‚úÖ "sindresorhus" (consistent online identity)
- ‚ùå "xX_C0d3r_Xx" (looks unprofessional)

### 3. Bio

Your bio should be concise and informative:

```
[Your role/title] | [Technologies/Interests] | [Location (optional)]
```

**Good examples:**
- "Full-stack developer. Open source enthusiast. Rust & TypeScript."
- "Student @Stanford. Building tools for developers."
- "Backend engineer @Company. Maintainer of @project."

**Bad examples:**
- "Just a guy who codes" (too vague)
- "10x developer and thought leader" (cringe)
- "" (empty bio = suspicious)

### 4. Location

Optional, but helpful for:
- Time zone awareness
- Local community connections
- Conference/meetup opportunities

### 5. Website/Links

Add links to:
- Personal website or blog
- Twitter/X profile
- LinkedIn (optional)

## Pinned Repositories

GitHub lets you pin up to 6 repositories to your profile. Choose wisely.

### What to Pin

<CardGrid>
<Card title="Your Best Work" icon={<Star className="h-5 w-5 text-emerald-400" />}>
Projects that showcase your skills. Quality over quantity.
</Card>

<Card title="Open Source Contributions" icon={<GitPullRequest className="h-5 w-5 text-emerald-400" />}>
Forks where you've made significant contributions.
</Card>

<Card title="Side Projects" icon={<Code className="h-5 w-5 text-emerald-400" />}>
Interesting projects that show your interests.
</Card>

<Card title="Documentation/Guides" icon={<FileText className="h-5 w-5 text-emerald-400" />}>
Written content showing communication skills.
</Card>
</CardGrid>

### What NOT to Pin

- Empty or barely-started repos
- Course homework/tutorials
- Forks you never modified
- Private joke projects

### How to Pin

1. Go to your profile page
2. Click "Customize your pins"
3. Select up to 6 repositories
4. Drag to reorder

## Contribution Graph

Your contribution graph (the green squares) tells a story.

### What Maintainers See

| Pattern | Interpretation |
|---------|----------------|
| Consistent activity | Reliable, committed |
| Sporadic bursts | May not follow through |
| Hacktoberfest spikes only | Contribution farmer? |
| Recent start | New to open source (neutral) |
| Empty | No public activity (yellow flag) |

### How to Build Your Graph

<Checklist>
<ChecklistItem>Make commits on public repositories (private don't count)</ChecklistItem>
<ChecklistItem>Commit to your fork's main/master branch or open PRs</ChecklistItem>
<ChecklistItem>Configure your Git email to match your GitHub email</ChecklistItem>
<ChecklistItem>Aim for consistency over intensity</ChecklistItem>
</Checklist>

<Callout type="warning" title="Don't Game the System">
Fake commits and contribution inflation are obvious and will hurt your reputation. Real, quality contributions always win.
</Callout>

## Social Proof

### Organizations

Join organizations you contribute to. They appear on your profile and add credibility.

### Achievements

GitHub has achievement badges. Some notable ones:
- **Pull Shark** ‚Äî Merged PRs
- **YOLO** ‚Äî Merged PR without review
- **Starstruck** ‚Äî Created a repo with 16+ stars
- **Pair Extraordinaire** ‚Äî Co-authored commits

### Sponsors

If you're eligible, set up GitHub Sponsors. Even if no one sponsors you, it shows you're serious about open source.
