# Becoming a Long-Term Contributor

Moving beyond occasional contributions to become a valued community member.

## The Journey

```
First PR → Regular Contributor → Trusted Contributor → Core/Maintainer
```

This journey typically takes 6-18 months of consistent contribution.

## What Sets Long-Term Contributors Apart

<CardGrid>
<Card title="Consistency">
Regular contributions over time, not bursts of activity.
</Card>

<Card title="Reliability">
Following through on commitments. Communicating when you can't.
</Card>

<Card title="Helpfulness">
Helping others in issues, reviews, and discussions.
</Card>

<Card title="Ownership">
Taking responsibility for areas of the codebase.
</Card>
</CardGrid>

## Building Reputation

Reputation is built through:

| Action | Impact |
|--------|--------|
| Consistent quality PRs | Shows skill and reliability |
| Reviewing others' PRs | Shows investment in project |
| Helping newcomers | Shows leadership |
| Triaging issues | Shows initiative |
| Writing documentation | Shows thoroughness |

<Callout type="tip">
Reviewing PRs is one of the fastest ways to build reputation. It shows you understand the codebase and care about quality.
</Callout>

## Path to Maintainership

<Warning>
Don't ask to become a maintainer. Demonstrate maintainer-level responsibility and it will be offered.
</Warning>

Maintainer responsibilities:
- Merging PRs
- Setting direction
- Mentoring contributors
- Handling releases
- Community management
