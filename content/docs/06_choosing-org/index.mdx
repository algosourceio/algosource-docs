# Choosing the Right Organization

Not all open source organizations are created equal. Here's how to find the right fit.

## What to Look For

<CardGrid>
<Card title="Active Community">
Regular commits, quick issue responses, active discussions.
</Card>

<Card title="Good Documentation">
Clear README, CONTRIBUTING guide, and code comments.
</Card>

<Card title="Welcoming Culture">
Constructive feedback, helpful maintainers, beginner-friendly labels.
</Card>

<Card title="Technical Fit">
Technologies you know or want to learn.
</Card>
</CardGrid>

## Red Flags

<Warning>
Walk away from projects that show these signs:
</Warning>

- No response to PRs for months
- Hostile or dismissive maintainers
- Abandoned with no recent activity
- No contribution guidelines

## Evaluation Checklist

<Checklist>
<ChecklistItem>Last commit within the past month</ChecklistItem>
<ChecklistItem>Issues get responses within a week</ChecklistItem>
<ChecklistItem>CONTRIBUTING.md exists and is helpful</ChecklistItem>
<ChecklistItem>Code of Conduct is present</ChecklistItem>
<ChecklistItem>CI/CD is set up and passing</ChecklistItem>
<ChecklistItem>Maintainers are respectful in discussions</ChecklistItem>
</Checklist>
